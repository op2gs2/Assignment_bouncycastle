1) 암호화 해시함수 개요

가) 해시함수 정의

- 임의의 길이의 메시지를 입력 받아, 고정된 길이의 해시 값으로 출력하는 함수이다.

나) 암호화 해시함수

- 역변환이 어려운 단방향(One-way) 해시함수로, 해시 값으로부터 입력 값을 알아내기 어려운 것을 말한다.

- 정보보호나 보안을 위해 사용된다.

다) 암호화 해시함수의 특성

- 역상 저항성(Pre-image resistance) : 해시 값으로부터 입력 값을 생성하는 것이 계산상 어려워야 한다.

- 2차 역상 저항성(Second pre-image resistance) : 해시 값 변경 없이 입력 값을 변경하는 것이 계산상 어려워야 한다.

- 충돌 저항성(Collision resistance) : 동일한 해시 값을 생성하는 2개의 입력 값을 찾는 것이 계산상 어려워야 한다.

라) 응용 분야

- SSL 인증이나 디지털 서명에 사용된다.

- 프로그램이나 파일의 변조 검출 : 사용 측에서 해시 값을 다시 계산하여, 원본 사이트에서 제공하는 해시 값과 비교한다.

- 패스워드 암호화 : 최초 입력된 패스워드의 해시 값만 저장하고, 추후 패스워드 확인 단계에서 입력된 패스워드의 해시 값을 다시 계산하여 저장되어 있는 해시 값과 동일한지 비교한다.

2) 암호화 해시 함수 종류

가) MD5

- 메시지를 512비트 블록 단위로 처리하며, 128비트의 해시 값을 출력한다.

- MD5는 설계상 결함이 발견되었고, 결함을 이용해 SSL 인증서를 변조하는 것이 가능하다는 것이 밝혀져, 보안 관련 용도로는 더 이상 권장되지 않는다.

나) SHA-1

- 가장 많이 사용되는 SHA(Secure Hash Algorithm) 중의 하나로, 160비트의 해시 값을 출력한다.

다) HAS-160

- 한국형 전자서명 표준인 KCDSA에서 사용할 목적으로 SHA-1을 기반으로 개발된 해시 함수이다.

- SHA-1과 마찬가지로 80단계의 연산을 거치며, 160비트의 해시 값을 출력한다.
